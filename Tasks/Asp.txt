HttpResponse

 - Middleware (промежуточное ПО) — это компонент, который обрабатывает HTTP-запрос на пути от клиента к серверу и обратно.
 
 - Все данные запроса передаются в middleware через объект Microsoft.AspNetCore.Http.HttpContext. Этот объект инкапсулирует информацию о запросе, позволяет управлять ответом и, кроме того, имеет еще много другой функциональности.
 
 - HttpContext используется на всём пути обработки запроса — от первого middleware до контроллера и обратно — и позволяет каждому этапу обработки видеть и при необходимости изменять как запрос, так и ответ.
 
HttpRequest
 
 - HttpRequest — это объект, который содержит всю информацию о запросе, пришедшем от клиента на сервер.
 
Controller

 - Controller — это специальный класс, который отвечает за обработку HTTP-запросов и возврат ответа клиенту.
 
Куда приходят данные

 - Если данные в URL (?id=5 или /user/5), они попадают в обычные параметры (например, int id)
 
 - Если данные в теле запроса (например, JSON), они приходят в объект с [FromBody]
 
  - Обмен данными произовдится при помощи Http протокола.
  
  - В обмен данных входит обьязательно запрос и ответ,как в принципе работает телефон.
  
  - Запрос(Request) состоит из : 
  
  - Protocol(Http/Https) - Http от Https отличается сертификатом безопасности,Https при передаче данных шифрует данные сертификатом безопасности.
  - Host
  - Port
  - Route
  - Methods
 
  - Body
  - Headers
  - Query params - для передаче Query params ставится знак вопроса после Route,и параметры передаются как названия_параметра=значение_параметра.Если параметра нету,то между ними ставится знак &. Пример ?limit=10&skip=0
  
 Примеры
 http://localhost:5151/task
 Protocol -  http://
 Host - localhost
 Port - 5151
 Route - task
 Query params - тут нету
 
 Для того чтобы получить Body,Headers,Query params используется специальные атрибуты [FromBody],[FromHeader(Name="Content-Type")],[FromQuery].
